# .NET 桌面应用发布技能

## 角色定义

你是一个专业的 .NET 桌面应用发布工程师，负责执行标准化的版本发布流程。
你必须严格按照以下步骤执行，每一步完成后报告状态，遇到错误立即停止并报告。

---

## 发布流程

当用户说"发布版本"或类似指令时，按以下步骤严格顺序执行：

### 第一步：环境检查

执行以下命令，确认开发环境就绪：

```shell
dotnet --version
```

确认输出为 6.x / 8.x / 10.x 其中之一。如果命令失败，提示用户安装 .NET SDK。

检查当前项目根目录下是否存在 `publish-config.json`，如果不存在，
引导用户创建（参考本技能末尾的配置模板）。

### 第二步：读取发布配置

读取项目根目录下的 `publish-config.json` 文件，获取以下关键信息：

- `projectPath`：.csproj 项目文件路径
- `appName`：应用名称
- `currentVersion`：当前版本号
- `publishProfiles`：发布目标配置数组
- `outputBaseDir`：发布输出根目录

### 第三步：确定新版本号

读取 `publish-config.json` 中的 `currentVersion` 字段（格式为 `x.y.z`）。

向用户确认版本升级方式：
- **patch**（修订号 +1）：bug 修复，如 1.0.0 → 1.0.1
- **minor**（次版本号 +1）：新功能，如 1.0.0 → 1.1.0
- **major**（主版本号 +1）：重大变更，如 1.0.0 → 2.0.0
- **自定义**：用户直接指定版本号

如果用户指令中已经包含版本号（如"发布 v2.1.0"），则直接使用该版本号。
如果用户说"发布版本"但未指定类型，默认使用 **patch**。

### 第四步：更新版本号到项目文件

将新版本号同步写入以下位置：

1. **`publish-config.json`** 中的 `currentVersion` 字段
2. **.csproj 项目文件**中的版本属性（如果不存在则添加到第一个 PropertyGroup 中）：

```xml
<PropertyGroup>
    <Version>新版本号</Version>
    <FileVersion>新版本号.0</FileVersion>
    <AssemblyVersion>新版本号.0</AssemblyVersion>
</PropertyGroup>
```

### 第五步：编写更新日志

询问用户本次更新的主要内容，然后将新版本的更新日志**追加插入**到
`CHANGELOG.md` 文件顶部（在标题行之后），格式如下：

```markdown
## [版本号] - 日期(YYYY-MM-DD)

### 新增
- 功能描述

### 修复
- 修复描述

### 变更
- 变更描述
```

如果 `CHANGELOG.md` 不存在，则创建该文件，顶部标题为 `# 更新日志`。

如果用户说"没什么特别的"或类似表述，则使用以下默认内容：
```markdown
## [版本号] - 日期
### 变更
- 常规更新与优化
```

### 第六步：构建与发布

读取 `publish-config.json` 中的 `publishProfiles` 数组，
对每一个发布配置执行 `dotnet publish` 命令。

对于每个 profile，拼接命令��下：

```shell
dotnet publish "{projectPath}" \
  -c Release \
  -r {rid} \
  -f {framework} \
  --self-contained {selfContained} \
  -p:PublishSingleFile={singleFile} \
  -p:PublishTrimmed={trimmed} \
  -p:IncludeNativeLibrariesForSelfExtract=true \
  -p:EnableCompressionInSingleFile=true \
  -p:DebugType=none \
  -p:DebugSymbols=false \
  -o "{outputBaseDir}/{appName}-v{version}-{rid}"
```

注意事项：
- Windows 上命令用 `^` 或写成单行，不要用 `\` 换行
- 如果 `trimmed` 为 true，追加 `-p:TrimMode=partial`（部分裁剪，兼容性更好）
- 如果 profile 中设置了 `readyToRun` 为 true，追加 `-p:PublishReadyToRun=true`
- 发布前先清理旧的输出目录

**每个 profile 发布完成后**，报告：
- ✅ 成功：profile 名称、输出路径、主要 exe 文件大小
- ❌ 失败：错误信息

### 第七步：打包为 ZIP

对每个成功发布的输出目录，使用 PowerShell 打包为 ZIP：

```powershell
Compress-Archive -Path "{输出目录}\*" -DestinationPath "{outputBaseDir}\{appName}-v{version}-{rid}.zip" -Force
```

### 第八步：生成发布摘要

所有步骤完成后，输出一份结构化的发布摘要：

```
══════════════════════════════════════════
  发布完成摘要
══════════════════════════════════════════
  应用名称：{appName}
  发布版本：v{version}
  发布时间：{当前时间}
──────────────────────────────────────────
  发布产物：
  ✅ {appName}-v{version}-win-x64.zip     (XX MB)
  ✅ {appName}-v{version}-win-x86.zip     (XX MB)
  ❌ {appName}-v{version}-win-arm64       (构建失败)
──────────────────────────────────────────
  文件位置：{outputBaseDir}
  更新日志：已更新 CHANGELOG.md
══════════════════════════════════════════
```

## 发布配置模板

如果项目中不存在 `publish-config.json`，使用以下模板创建：

```json
{
  "appName": "MyApp",
  "currentVersion": "1.0.0",
  "projectPath": "src/MyApp/MyApp.csproj",
  "outputBaseDir": "publish",
  "publishProfiles": [
    {
      "name": "win-x64",
      "rid": "win-x64",
      "framework": "net8.0-windows",
      "selfContained": true,
      "singleFile": true,
      "trimmed": false,
      "readyToRun": false
    },
    {
      "name": "win-x86",
      "rid": "win-x86",
      "framework": "net8.0-windows",
      "selfContained": true,
      "singleFile": true,
      "trimmed": false,
      "readyToRun": false
    }
  ]
}
```

### 配置字段说明

| 字段 | 说明 |
|---|---|
| `appName` | 应用名称，用于输出文件命名 |
| `currentVersion` | 当前版本号，每次发布自动更新 |
| `projectPath` | .csproj 文件相对路径 |
| `outputBaseDir` | 发布输出根目录 |
| `rid` | 运行时标识符：win-x64 / win-x86 / win-arm64 |
| `framework` | 目标框架：net6.0-windows / net8.0-windows / net10.0-windows |
| `selfContained` | 自包含发布（true=不需要用户装运行时，兼容性最好） |
| `singleFile` | 单文件发布（true=打包为一个 exe） |
| `trimmed` | IL 裁剪（true=减小体积，但可能有兼容问题） |
| `readyToRun` | 预编译（true=启动更快，但文件更大） |

---

## Win7 兼容性说明

如果需要兼容 Windows 7：

1. 使用 `net6.0-windows` 框架（.NET 6 是最后官方支持 Win7 的版本）
2. 必须设置 `selfContained: true`
3. 目标用户的 Win7 需要安装：
   - KB2533623 更新补丁
   - KB3063858 更新补丁（部分情况）
4. 在 .csproj 中不要使用 `<WindowsSdkPackageVersion>`

如果用户明确说需要兼容 Win7，自动在 publishProfiles 中将 framework 
调整为 `net6.0-windows`，并提醒上述注意事项。

---

## 异常处理规则

1. **dotnet 未安装**：提示安装链接 https://dotnet.microsoft.com/download
2. **构建失败**：输出完整错误日志，不继续后续 profile 的构建
3. **版本号格式错误**：必须符合 semver 格式 x.y.z
4. **项目文件不存在**：提示检查 projectPath 配置
5. **磁盘空间不足**：发布前检查磁盘可用空间，低于 1GB 时警告
6. **部分 profile 失败**：继续执行其他 profile，最终摘要中标注失败项

    <ApplicationIcon>app.ico</ApplicationIcon>

    <DebugType>none</DebugType>
    <DebugSymbols>false</DebugSymbols>

    <!-- 启用最新C#语言特性 -->
    <LangVersion>latest</LangVersion>
    <Nullable>enable</Nullable>

    <!-- 单文件发布时嵌入原生库 -->
    <IncludeNativeLibrariesForSelfExtract>true</IncludeNativeLibrariesForSelfExtract>
  </PropertyGroup>

  <!-- 嵌入版本信息到程序中（可在代码中读取） -->
  <ItemGroup>

  </ItemGroup>

## 3. 备用 PowerShell 发布脚本

`scripts/publish.ps1`（作为 Skill 失效时的备用方案）

```powershell
#Requires -Version 5.1
<#
.SYNOPSIS
    .NET 桌面应用自动发布脚本
.DESCRIPTION
    读取 publish-config.json 配置，自动完成版本更新、构建、发布、打包
.PARAMETER VersionBump
    版本升级方式: patch | minor | major | 自定义版本号
.PARAMETER SkipZip
    跳过ZIP打包
.EXAMPLE
    .\scripts\publish.ps1
    .\scripts\publish.ps1 -VersionBump minor
    .\scripts\publish.ps1 -VersionBump "2.1.0"
#>

param(
    [Parameter(Position = 0)]
    [string]$VersionBump = "patch",
    
    [switch]$SkipZip,
    [switch]$SkipChangelog
)

Set-StrictMode -Version Latest
$ErrorActionPreference = "Stop"

# ============================================================
# 工具函数
# ============================================================

function Write-Header($text) {
    $line = "=" * 56
    Write-Host "`n$line" -ForegroundColor Cyan
    Write-Host "  $text" -ForegroundColor Cyan
    Write-Host "$line" -ForegroundColor Cyan
}

function Write-Step($step, $text) {
    Write-Host "`n[$step] $text" -ForegroundColor Yellow
}

function Write-Success($text) {
    Write-Host "  ✅ $text" -ForegroundColor Green
}

function Write-Fail($text) {
    Write-Host "  ❌ $text" -ForegroundColor Red
}

function Get-BumpedVersion([string]$current, [string]$bump) {
    # 尝试解析为自定义版本号
    if ($bump -match '^\d+\.\d+\.\d+$') {
        return $bump
    }
    
    $parts = $current.Split('.')
    $major = [int]$parts[0]
    $minor = [int]$parts[1]
    $patch = [int]$parts[2]
    
    switch ($bump.ToLower()) {
        "major" { $major++; $minor = 0; $patch = 0 }
        "minor" { $minor++; $patch = 0 }
        "patch" { $patch++ }
        default { 
            Write-Fail "未知的版本升级方式: $bump"
            exit 1
        }
    }
    
    return "$major.$minor.$patch"
}

function Get-FileSizeMB($path) {
    if (Test-Path $path) {
        $size = (Get-Item $path).Length / 1MB
        return "{0:N1} MB" -f $size
    }
    return "N/A"
}

# ============================================================
# 主流程
# ============================================================

$projectRoot = Split-Path -Parent $PSScriptRoot
Push-Location $projectRoot

try {
    Write-Header ".NET 桌面应用发布工具"
    
    # ----------------------------------------------------------
    # 第一步：环境检查
    # ----------------------------------------------------------
    Write-Step 1 "环境检查"
    
    $dotnetVersion = & dotnet --version 2>$null
    if (-not $dotnetVersion) {
        Write-Fail "未检测到 .NET SDK，请先安装: https://dotnet.microsoft.com/download"
        exit 1
    }
    Write-Success ".NET SDK: $dotnetVersion"
    
    # ----------------------------------------------------------
    # 第二步：读取配置
    # ----------------------------------------------------------
    Write-Step 2 "读取发布配置"
    
    $configPath = Join-Path $projectRoot "publish-config.json"
    if (-not (Test-Path $configPath)) {
        Write-Fail "未找到 publish-config.json，请先创建配置文件"
        exit 1
    }
    
    $config = Get-Content $configPath -Raw -Encoding UTF8 | ConvertFrom-Json
    $appName = $config.appName
    $currentVersion = $config.currentVersion
    $projectPath = Join-Path $projectRoot $config.projectPath
    $outputBaseDir = Join-Path $projectRoot $config.outputBaseDir
    
    Write-Success "应用: $appName  当前版本: v$currentVersion"
    
    # ----------------------------------------------------------
    # 第三步：计算新版本号
    # ----------------------------------------------------------
    Write-Step 3 "版本升级"
    
    $newVersion = Get-BumpedVersion $currentVersion $VersionBump
    Write-Success "v$currentVersion → v$newVersion ($VersionBump)"
    
    # ----------------------------------------------------------
    # 第四步：更新版本号
    # ----------------------------------------------------------
    Write-Step 4 "更新版本号"
    
    # 更新 publish-config.json
    $config.currentVersion = $newVersion
    $config | ConvertTo-Json -Depth 10 | Set-Content $configPath -Encoding UTF8
    Write-Success "publish-config.json 已更新"
    
    # 更新 .csproj
    if (Test-Path $projectPath) {
        $csprojContent = Get-Content $projectPath -Raw -Encoding UTF8
        
        # 替换或插入版本号
        $versionProps = @{
            'Version'         = $newVersion
            'FileVersion'     = "$newVersion.0"
            'AssemblyVersion' = "$newVersion.0"
        }
        
        foreach ($prop in $versionProps.GetEnumerator()) {
            $pattern = "(<$($prop.Key)>)(.*?)(</$($prop.Key)>)"
            if ($csprojContent -match $pattern) {
                $csprojContent = $csprojContent -replace $pattern, "`${1}$($prop.Value)`${3}"
            }
        }
        
        Set-Content $projectPath $csprojContent -Encoding UTF8
        Write-Success "$($config.projectPath) 已更新"
    }
    
    # ----------------------------------------------------------
    # 第五步：更新日志
    # ----------------------------------------------------------
    if (-not $SkipChangelog) {
        Write-Step 5 "更新日志"
        
        $changelogPath = Join-Path $projectRoot "CHANGELOG.md"
        $date = Get-Date -Format "yyyy-MM-dd"
        $entry = @"

## [$newVersion] - $date

### 变更
- 常规更新与优化

"@
        
        if (Test-Path $changelogPath) {
            $content = Get-Content $changelogPath -Raw -Encoding UTF8
            # 在第一行标题之后插入
            $content = $content -replace "(# .+\r?\n)", "`$1$entry"
            Set-Content $changelogPath $content -Encoding UTF8
        }
        else {
            $content = "# 更新日志`n$entry"
            Set-Content $changelogPath $content -Encoding UTF8
        }
        
        Write-Success "CHANGELOG.md 已更新"
    }
    
    # ----------------------------------------------------------
    # 第六步：构建与发布
    # ----------------------------------------------------------
    Write-Step 6 "构建与发布"
    
    # 确保输出目录存在
    if (-not (Test-Path $outputBaseDir)) {
        New-Item -ItemType Directory -Path $outputBaseDir -Force | Out-Null
    }
    
    $results = @()
    
    foreach ($profile in $config.publishProfiles) {
        $rid = $profile.rid
        $profileName = $profile.name
        $outDir = Join-Path $outputBaseDir "$appName-v$newVersion-$rid"
        $zipPath = Join-Path $outputBaseDir "$appName-v$newVersion-$rid.zip"
        
        Write-Host "`n  📦 构建 $profileName ..." -ForegroundColor White
        
        # 清理旧输出
        if (Test-Path $outDir) { Remove-Item $outDir -Recurse -Force }
        if (Test-Path $zipPath) { Remove-Item $zipPath -Force }
        
        # 拼接 dotnet publish 参数
        $publishArgs = @(
            "publish"
            "`"$projectPath`""
            "-c", "Release"
            "-r", $rid
            "-f", $profile.framework
            "--self-contained", $profile.selfContained.ToString().ToLower()
            "-p:PublishSingleFile=$($profile.singleFile.ToString().ToLower())"
            "-p:IncludeNativeLibrariesForSelfExtract=true"
            "-p:EnableCompressionInSingleFile=true"
            "-p:DebugType=none"
            "-p:DebugSymbols=false"
            "-o", "`"$outDir`""
        )
        
        if ($profile.trimmed -eq $true) {
            $publishArgs += "-p:PublishTrimmed=true"
            $publishArgs += "-p:TrimMode=partial"
        }
        
        if ($profile.readyToRun -eq $true) {
            $publishArgs += "-p:PublishReadyToRun=true"
        }
        
        # 执行发布
        $process = Start-Process -FilePath "dotnet" -ArgumentList $publishArgs `
            -NoNewWindow -Wait -PassThru `
            -RedirectStandardOutput "$outDir.build.log" `
            -RedirectStandardError "$outDir.error.log"
        
        $result = @{
            Name    = $profileName
            Rid     = $rid
            OutDir  = $outDir
            ZipPath = $zipPath
            Success = ($process.ExitCode -eq 0)
        }
        
        if ($result.Success) {
            # 查找 exe 文件大小
            $exeFile = Get-ChildItem $outDir -Filter "*.exe" | Select-Object -First 1
            $exeSize = if ($exeFile) { Get-FileSizeMB $exeFile.FullName } else { "N/A" }
            $result.ExeSize = $exeSize
            
            Write-Success "$profileName 构建成功 (exe: $exeSize)"
            
            # 打包 ZIP
            if (-not $SkipZip) {
                Compress-Archive -Path "$outDir\*" -DestinationPath $zipPath -Force
                $zipSize = Get-FileSizeMB $zipPath
                $result.ZipSize = $zipSize
                Write-Success "$profileName 已打包 → $($result.ZipPath | Split-Path -Leaf) ($zipSize)"
            }
        }
        else {
            $errorContent = ""
            if (Test-Path "$outDir.error.log") {
                $errorContent = Get-Content "$outDir.error.log" -Raw
            }
            Write-Fail "$profileName 构建失败"
            if ($errorContent) {
                Write-Host "    错误信息: $errorContent" -ForegroundColor Red
            }
        }
        
        # 清理临时日志
        Remove-Item "$outDir.build.log" -ErrorAction SilentlyContinue
        Remove-Item "$outDir.error.log" -ErrorAction SilentlyContinue
        
        $results += $result
    }
    
    # ----------------------------------------------------------
    # 第七步：发布摘要
    # ----------------------------------------------------------
    Write-Host ""
    Write-Header "发布完成摘要"
    Write-Host "  应用名称：$appName"
    Write-Host "  发布版本：v$newVersion"
    Write-Host "  发布时间：$(Get-Date -Format 'yyyy-MM-dd HH:mm:ss')"
    Write-Host "  $("-" * 52)" -ForegroundColor DarkGray
    Write-Host "  发布产物：" 
    
    foreach ($r in $results) {
        if ($r.Success) {
            $zipFile = $r.ZipPath | Split-Path -Leaf
            $size = if ($r.ZipSize) { $r.ZipSize } else { $r.ExeSize }
            Write-Success "$zipFile  ($size)"
        }
        else {
            Write-Fail "$($r.Name) 构建失败"
        }
    }
    
    Write-Host "  $("-" * 52)" -ForegroundColor DarkGray
    Write-Host "  文件位置：$outputBaseDir"
    Write-Host ""
    
    $successCount = ($results | Where-Object { $_.Success }).Count
    $totalCount = $results.Count
    
    if ($successCount -eq $totalCount) {
        Write-Host "  🎉 全部 $totalCount 个目标构建成功！" -ForegroundColor Green
    }
    else {
        Write-Host "  ⚠️  $successCount/$totalCount 个目标构建成功" -ForegroundColor Yellow
    }
    
    Write-Host ""
}
finally {
    Pop-Location
}
```

## 4. 发布配置文件

`publish-config.json`

```json

## 6. 使用方式

在 Trae 对话框中直接输入：

| 你说 | AI 做什么 |
|---|---|
| `发布版本` | patch 升级 + 构建 + 打包 |
| `发布 minor 版本` | minor 升级 + 构建 + 打包 |
| `发布 v2.0.0` | 指定版本号 + 构建 + 打包 |
| `初始化发布配置` | 创建 `publish-config.json` |
| `查看当前版本` | 读取并显示当前版本信息 |

需要兼容 Win7 时，将配置中的 `framework` 改为 `net6.0-windows`，对话中告诉 AI **"需要兼容 Win7"** 即可自动调整。

> **核心思路**：所有可变参数收敛到 `publish-config.json`，Skill 读取配置驱动发布流程，一次配置、反复使用，版本号由 Skill 自动递增写回。
